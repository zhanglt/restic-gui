问题：缺少 moc 文件导致链接错误
原因：修改了测试文件但没有重新运行 qmake

════════════════════════════════════════════════════════════
  解决方法（3选1）
════════════════════════════════════════════════════════════

【方法 1】使用批处理脚本（最简单）★推荐★
----------------------------------------
在 tests 目录下运行：
  rebuild.bat

脚本会自动完成所有步骤。


【方法 2】使用 Qt Creator（最方便）
----------------------------------------
1. 用 Qt Creator 打开 tests/tests.pro
2. 右键项目 → Run qmake
3. 右键项目 → Clean All
4. 右键项目 → Rebuild All


【方法 3】手动命令行
----------------------------------------
在 Qt 命令提示符中执行：

  cd E:\test\qt\restic-gui\tests
  qmake tests.pro
  nmake clean
  nmake
